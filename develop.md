你是一名资深 Python 桌面应用开发工程师，精通 PySide6 与 Windows 平台特性。请帮我搭建一个名为 **FlashTranslator** 的 Windows 桌面划词翻译器初始脚手架。

## 一、核心功能需求
1. **划词触发**：用户用鼠标选中文本后，按下全局快捷键（默认 Ctrl+Q），在鼠标光标位置弹出半透明悬浮翻译框。
2. **悬浮翻译窗**：
   - 无边框、置顶、半透明（可调）、带圆角和投影。
   - 显示翻译结果，5秒无操作自动隐藏，也可手动关闭。
   - 窗口位置跟随鼠标，并智能避免超出屏幕边界。
3. **语言自动反向**：设置默认目标语言（如中文）。检测源语言，若源语言与目标相同则自动反向翻译（选中英文→译中，选中中文→译英）。
4. **多引擎可插拔架构**：
   - 免费在线引擎（示例：MyMemory，无需 API Key）
   - 付费大模型 API（兼容 OpenAI/DeepSeek 等），可配置系统角色提示词实现口语化/学术化风格
   - 本地大模型（通过 Ollama 或 llama-cpp-python 加载），同样支持角色设定
5. **系统托盘**：最小化到托盘，右键菜单可切换引擎、打开设置、退出。
6. **设置界面**：可修改目标语言、选择引擎、填写 API Key 和角色提示词、指定本地模型路径等。

## 二、技术栈与要求
- **OS**：Windows 10/11 优先，代码结构预留 Linux 扩展性（路径处理、热键库选择等）。
- **GUI**：PySide6（Qt 6.5+），使用 QSS 美化界面。
- **Python**：3.10+，全程使用 type hints，数据类用 `dataclasses`。
- **并发**：所有翻译操作必须在 `QThread` 工作线程中执行，通过信号与槽返回结果，绝不允许阻塞主线程。
- **全局热键**：使用 `keyboard` 库注册热键。注意：在 Windows 上需要**管理员权限**才能捕获全局按键，这是当前方案的已知限制，请在注释中说明。
- **划词获取**：
  - 通过模拟 Ctrl+C 复制选中文本，使用 `pyperclip` 读取剪贴板。
  - 获取后必须**还原剪贴板**原内容，并考虑 Windows 剪贴板历史（云剪贴板）的干扰，加入适当延时和异常处理。
- **日志**：使用 `logging`，同时输出到 `logs/app.log` 和控制台，记录关键操作和异常。
- **配置持久化**：使用 JSON 文件存储设置，通过 `dataclass` 和自定义序列化器读写。

## 四、详细设计规范
1. **信号与槽**：各引擎翻译完成后必须发射 `result_ready(str)` 信号，悬浮窗连接该信号并显示结果。若翻译出错则发射 `error_occurred(str)` 信号。
2. **线程安全**：子线程中严禁直接访问或修改 UI 控件，所有通信均通过信号。
3. **依赖注入**：主程序创建配置实例和引擎实例，注入到需要它们的 UI 组件中。
4. **异常处理**：每个引擎的 `translate` 方法必须捕获所有异常，转为自定义的 `TranslationError` 异常并记录日志，由 UI 层展示友好错误信息。
5. **类型标注**：所有函数和方法必须使用完整的 Python type hints。
6. **注释**：关键逻辑用中文详细注释，特别是 Windows 特有处理（如热键权限、剪贴板还原延时）。
